Если в некоторых частях case не будет оператора break, как в приведённом вами примере, выполнение кода будет продолжаться до тех пор, пока не встретится первый break или конец блока switch. 
Это может привести к выполнению нескольких блоков case подряд, даже если они не соответствуют значению переменной в switch.

switch (value) {
    case 1:
        i = 1;
        break;
    case 2:
        i = 2;   // Здесь нет break, выполнение продолжится в следующий case
    case 3:
        i = 3;   // Выполняется этот код, потому что в case 2 не было break
        break;
}

Если value == 2 выполнится код в case 2 (i = 2), но поскольку нет break, выполнение продолжится в case 3.
Затем выполнится код из case 3 (i = 3), и только потом сработает break.
Результат: переменной i сначала присвоится значение 2, а затем оно будет перезаписано значением 3

Если не указать break в каком-то из блоков case, выполнение перейдет в следующий case, даже если он не соответствует значению переменной. В результате могут выполняться несколько блоков кода подряд.
